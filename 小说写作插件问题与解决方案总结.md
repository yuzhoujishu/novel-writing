## 问题一：UUID 依赖缺失

### 问题描述

执行 `node scripts/lib/state-manager.js active` 时报错：

```
Error: Cannot find module 'uuid'
```

`state-manager.js` 内部依赖了 `uuid` 模块，但插件目录中未安装该依赖。

### 解决方案

在插件根目录下执行 npm 安装：

```bash
cd C:\Users\Administrator\.claude\plugins\cache\novel-writing\novel-writing\1.0.0
npm install uuid
```

安装后验证：`node scripts/lib/state-manager.js active` 正常输出。

## 问题二：路径错误 — `skills/` vs `novel-writing/`

### 问题描述

Claude Code 会话加载的 Skill 路径为 `skills/auto/scripts/lib/state-manager.js`，但实际脚本位于 `novel-writing/1.0.0/scripts/lib/state-manager.js`。

两种路径的根本区别：

| 路径前缀 | 指向 |
|---------|------|
| `skills/auto/` | Skill 定义的描述文件所在目录，不含实际脚本 |
| `novel-writing/1.0.0/` | 插件实际安装目录，脚本和数据都在此处 |

### 解决方案

始终使用插件实际安装路径：

```
C:\Users\Administrator\.claude\plugins\cache\novel-writing\novel-writing\1.0.0\scripts\lib\state-manager.js
```

注意：Skill 描述文件中的 `scripts/lib/state-manager.js` 是相对路径，需拼接正确的插件根目录。

## 问题三：项目不存在 — 多次调用 skill 均返回 `activeProject: null`

### 问题描述

用户连续多次调用 `/novel-writing:auto --batch=5`，每次返回：

```json
{ "activeProject": null }
```

原因：插件的 `active` 命令不会自动创建项目，只查询已有项目。

### 解决方案

使用 `state-manager.js` 手动创建完整项目，包括所有状态文件：

```bash
# 1. 创建项目
node scripts/lib/state-manager.js create "星际迷途"

# 2. 加载项目
node scripts/lib/state-manager.js load a68901c9-7ae8-4b78-ae00-e8473cf0017b
```

**仅执行 create/load 不够** — 还需要手动创建以下数据文件（`state-manager.js` 不自动生成这些）：

- `outline.json` — 章节大纲结构
- `characters.json` — 角色信息
- `world.json` — 世界观设定
- `chapters/*.md` — 各章节内容文件

直接向 `data/{projectId}/` 目录写入 JSON/Markdown 文件。

## 问题四：Write 工具限制 — "File has not been read yet"

### 问题描述

使用 `Write` 工具覆盖已有文件时报错：

```
File has not been read yet. Read it first before writing to it.
```

Claude Code 要求在写入任何已有文件前，必须先用 `Read` 工具读取过。

### 解决方案

更新任何状态文件的标准流程：

```
1. Read 目标文件
2. Edit / Write 目标文件
```

对于 `outline.json`、`meta.json` 等频繁更新的文件，每次修改前先 `Read` 再 `Edit`。
